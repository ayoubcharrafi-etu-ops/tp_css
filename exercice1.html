Exercice 1 – Sélecteurs et spécificité

1.1 – Les sélecteurs CSS


<nav class="main-nav">
  <ul>
    <li><a href="/" class="active">Accueil</a></li>
    <li><a href="/about">À propos</a></li>
    <li><a href="/contact">Contact</a></li>
  </ul>
</nav>

<main>
  <article class="post featured">
    <h2>Premier article</h2>
    <p>Contenu...</p>
  </article>

  <article class="post">
    <h2>Deuxième article</h2>
    <p>Contenu...</p>
  </article>
</main>

1. Sélectionnez tous les liens dans la navigation

.main-nav a {
}

2. Sélectionnez uniquement le lien actif

.main-nav a.active {   
}

3. Sélectionnez l'article qui a la classe featured

article.post.featured {
}

4. Sélectionnez le premier paragraphe de chaque article

article p:first-child {
}

5. Sélectionnez les éléments <li> pairs

li:nth-child(even) {
}

1.2 – La spécificité

Classez ces sélecteurs par spécificité croissante :

Ordre (du moins fort au plus fort) Sélecteur Score (ID, Classes, Balises) Explication
1 — Le moins fort p 0, 0, 1 Juste une balise
2 .text 0, 1, 0 Une classe
3 p.text 0, 1, 1 Une classe + une balise
4 #main 1, 0, 0 Un ID (très fort !)
5 — Le plus fort #main .text p 1, 1, 1 ID + classe + balise

Si deux règles ont la même spécificité, laquelle gagne ?

/* Règle 1 */
p { color: red; }

/* Règle 2 */
p { color: blue; }


--> Le paragraphe sera bleu, car cette règle est en dernier
C'est la règle écrite en dernier dans le fichier CSS qui gagne. C'est la "cascade" dans CSS (Cascading Style Sheets).

Pourquoi faut-il éviter !important ?

/* À ÉVITER ! */
p { color: red !important; }

/* Cette règle sera ignorée même si elle est plus spécifique */
#main p { color: blue; }  /* → le paragraphe reste rouge malgré l'ID ! */
⚠️ Problème : !important écrase TOUTES les règles et rend le CSS très difficile à modifier. Si vous avez besoin de !important, c'est souvent signe que votre CSS est mal organisé. Mieux vaut augmenter la spécificité naturellement.
